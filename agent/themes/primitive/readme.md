# primitive (aka primitive-light)

Brand-neutral Inter-based theme. Use when demonstrating raw component primitives without brand pollution — documentation pages, agent introspection demos, baselines for screenshots.

## When to use

- The `/components` Storybook gallery
- Spec sheets, ADRs, internal docs that shouldn't carry Lore brand weight
- Onboarding demos where the brand should fade so the *system* is visible

## Anti-pattern

Don't use `primitive` in customer-facing product surfaces. The brand-neutral neutrality reads as missing brand, not as restraint. Use `lore-light` or `lore-dark` for product.

## Pairs with

`primitive-dark` (stub).
