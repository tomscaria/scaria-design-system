# Messaging — what we say at each surface

> Section-by-section copy rules. Builds on `voice.md` (tone) + `positioning.md` (substance).

## Hero (landing, deck cover)

- **Headline:** 6–10 words. Declarative. No questions.
- **Subhead:** 1 sentence, ≤ 20 words. Clarifies the headline with one concrete detail.
- **No paragraph copy.** Hero carries via type + visual, not narrative.
- **CTA:** 2–3 words. Verb + object.

Templates:
- `[Concrete noun]. [Present-tense verb]. [Measurable outcome].`
- `[Noun phrase that names the wedge]. [One-sentence what it does].`

## Product cards (max 3 per surface)

- Title: 2–4 words, noun-led
- Body: 1–2 sentences — what it does + one proof point
- Don't pad with adjectives ("powerful", "robust", "scalable" — banned)

## Stage / demo viewport (e.g. the deck's interactive section)

- Tab labels: 1–3 words. Action- or object-oriented.
- Bullets inside tabs: short fragments, ≤ 8 words each.
- Numbers always real or marked as illustrative.

## Proof (charts, flows, data)

- Chart titles: factual labels. "Monthly AUM growth," not "Explosive growth!"
- Annotations: numbers only, no adjectives.
- Flow descriptions: present tense, step-by-step.
  - "User taps Buy → Order fills onchain → Portfolio updates"

## Reputation / trust

- Leaderboard labels: name + metric. No commentary.
- Partner strip: logo-only preferred.
- Team: title + years of experience. Resist hagiography.

## CTA endcap

- Headline: ≤ 6 words. Imperative or declarative.
- Button: 2–3 words.
- No body copy. By here the argument is made.

## Length limits (hard)

| Element | Max words |
|---|---|
| Hero headline | 10 |
| Hero subhead | 20 |
| Card title | 4 |
| Card body | 30 |
| Tab label | 3 |
| Tab bullet | 8 |
| CTA headline | 6 |
| CTA button | 3 |
| Chart title | 5 |
| Metric label | 3 |

## Proof pairing rule

Every claim within visual proximity of its proof:

| Claim type | Required proof |
|---|---|
| Performance | Chart with time axis |
| AUM / volume | Live or recent number, dated |
| User count | Exact figure (not "thousands") |
| Security | Audit name + link |
| Team credibility | Title + years |

## Formatting

- Digits for all values ≥ 2; spell out "one" and "zero"
- Large numbers abbreviated: K / M / B
- Units always: "34 %" (space before %), "$1.2M", "1,200 wallets"
- Dates: relative preferred ("Monthly", "Every 30 days"); absolute as "Jan 2025"
- "onchain" one word lowercase; "DeFi" capitalized; "CEX" / "DEX" uppercase
